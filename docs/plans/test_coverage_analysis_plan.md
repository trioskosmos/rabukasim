# テストカバレッジ分析計画

## 目標
各カードのテスト数とタイプを分析し、追加テストの必要性を特定する。

## 分析項目

### 1. カード別テスト統計
- アビリティ数（トリガーごとのテスト数）
- シーケンスの長さ（各アビリティの複雑さ）
- デルタの数（検証の網羅性）

### 2. トリガータイプ別統計
- ONPLAY: プレイ時効果
- ONLIVESTART: ライブ開始時効果
- ONLIVESUCCESS: ライブ成功時効果
- ACTIVATED: アクティベート効果
- CONSTANT: 常時効果

### 3. テスト複雑さの分類
- **単純**: 1アビリティ、1シーケンス、1-2デルタ
- **普通**: 1-2アビリティ、2-3シーケンス、3-5デルタ
- **複雑**: 2+アビリティ、3+シーケンス、5+デルタ
- **モーダル**: 選択肢あり（追加テスト必要）

### 4. 追加テストが必要なパターン
1. **モーダル選択**: 複数オプションを持つカード
2. **条件付き効果**: 条件が満たされた/満たされない場合
3. **相手への効果**: 相手側の状態変化
4. **チェーン効果**: 複数ステージの効果

## 実装計画

### ステップ1: 分析スクリプト作成
`tools/analyze_test_coverage.py`を作成:
- semantic_truth_v3.jsonを読み込み
- 各カードの統計を収集
- レポートを生成

### ステップ2: レポート形式
```markdown
# テストカバレッジレポート

## サマリー
- 総カード数: XXX
- 総アビリティ数: XXX
- 平均アビリティ数/カード: X.X

## カード別統計

| カードID | アビリティ数 | トリガー | 複雑さ | 追加テスト必要 |
|---------|------------|---------|-------|--------------|
| PL!N-bp4-007-P | 3 | ONPLAY, CONSTANT, ONLIVESUCCESS | 普通 | いいえ |
| LL-PR-004-PR | 1 | ONLIVESTART | モーダル | はい (3オプション) |

## トリガー別統計
...
```

### ステップ3: 追加テストの自動生成
- モーダルオプションごとのテストケース生成
- 条件付き効果のポジティブ/ネガティブテスト生成
