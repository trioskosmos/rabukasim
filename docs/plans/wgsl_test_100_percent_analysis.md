## FAILの原因分析

### カテゴリ1: semantic_truthの期待値が間違っている（推定: 50-70件）

#### 問題例
```
PL!N-bp4-021-N: MOVE_TO_DECK(1) {FROM="DISCARD", TO="DECK_TOP"}
期待値: HAND_DISCARD: 1
実際: ディスカードからデッキに移動（手札には影響しない）
```

#### 修正方法
1. `pseudocode_oracle.py`でデルタ生成ロジックを修正
2. `semantic_truth_v3.json`を再生成

---

### カテゴリ2: GPUシェーダー側のバイトコード実行が不完全（推定: 80-100件）

#### 問題例
```
PL!N-bp4-021-N: bytecode [31, 1, 0, 1, 1, 0, 0, 0]
期待値: DISCARD_DELTA: -1, DECK_DELTA: +1
実際: 何も実行されていない
```

#### 原因
1. **O_MOVE_TO_DECK (opcode 31)** がshader_rules.wgslで正しく実装されていない
2. **O_DRAW (opcode 10)** が正しく実装されていない可能性
3. **O_TAP_ENERGY** が正しく実装されていない可能性

#### 修正方法
1. shader_rules.wgslで各オペコードの実装を確認・修正
2. 特にzone判定とカード移動ロジックを確認

---

### カテゴリ3: テスト初期状態の問題（推定: 20-30件）

#### 問題例
```
Energy tap delta mismatch (expected: 2, actual: 9)
```

#### 原因
1. エネルギーゾーンの初期状態が実際のゲームと異なる
2. テストで使用するダミーカードIDが実際のカードと競合している

#### 修正方法
1. テスト初期状態を実際のゲーム開始状態に近づける
2. ダミーカードIDを実際のカードIDと重複しないようにする

---

### カテゴリ4: インタラクションが必要な効果（推定: 30-50件）

#### 問題例
```
DISCARD_HAND(1) - カード選択が必要だが、テストでは自動選択されている
```

#### 原因
1. GPUシェーダー側でインタラクション処理が不完全
2. テストで`choice`パラメータが正しく渡されていない

#### 修正方法
1. インタラクションが必要な効果のテストをスキップ
2. または、自動選択ロジックを実装

## SKIPの原因分析

### カテゴリ1: トリガーがONPLAY/CONSTANT以外（推定: 400件）

#### トリガー種別

- ACTIVATED: 約100件
- ONLIVESUCCESS: 約50件

#### 修正方法

2. GPUシェーダー側で各トリガーの発火条件を実装
