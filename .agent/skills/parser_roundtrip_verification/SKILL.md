# Parser Roundtrip Verification Skill

This skill ensures that card abilities can be correctly parsed, compiled to bytecode, and then decompiled back into a readable format. This "roundtrip" verification is essential for catching parser regressions and ensuring bytecode integrity.

## 🎯 Purpose
The goal is to verify that **100% of compiled abilities** are valid and can be decompiled. If an ability fails the roundtrip, it indicates a bug in the `compiler/parser_v2.py` or the `decompile_bytecode.py` tool.

## 🛠️ Core Capability

### 1. Roundtrip Verification Script
Run the verification logic to check all cards in `data/cards_compiled.json`.
- **Command**: `uv run python .agent/skills/parser_roundtrip_verification/scripts/verify_roundtrip.py`
- **Report**: [parser_verification_report.json](file:///c:/Users/trios/.gemini/antigravity/vscode/loveca-copy/reports/parser_verification_report.json)

## 📋 Verification Logic
1. **Empty Bytecode Check**: Ensures that abilities with text actually generate bytecode (unless they are trigger-only).
2. **Decompilation Check**: Attempts to decompile the generated bytecode using `tools/decompile_bytecode.py`.
3. **Pattern Matching**: Flags "RETURN only" bytecode if the raw text contains effect keywords like `EFFECT:`, `DRAW`, etc.

## 🔄 Workflow for Parser Changes
When modifying `compiler/parser_v2.py` or `engine/models/ability.py`:
1. **Recompile**: `uv run python -m compiler.main`
2. **Verify Roundtrip**: Run this skill's verification script.
3. **Triage Errors**: Check the generated report for any cards that failed decompilation.
4. **Fix Parser**: Resolve the issues and repeat until 100% success.
